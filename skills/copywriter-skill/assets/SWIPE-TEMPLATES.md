# Structural Templates

Fill-in-the-blank templates derived from frameworks documented in reference files. Each template cites its source framework for deeper study.

---

## How to Use This File

1. Select template matching your content type
2. Fill in bracketed placeholders with your specifics
3. Consult cited reference section for application rules
4. Run Human-Check Protocol before finalizing

## 2. Sales Email Template

**Source:** Brunson's Hook-Story-Offer (MODERN-DIGITAL § 2.1)

```
SUBJECT LINE: [Curiosity hook or specific benefit]

---

[HOOK: 1-2 sentences that stop the scroll]
- Pattern interrupt OR
- Bold claim OR  
- Question they're already asking themselves

---

[STORY: 3-5 paragraphs]

Paragraph 1: Set the scene
"[Time marker], I was [situation they'll relate to]..."

Paragraph 2: The struggle
"I tried [common solutions]. None of them [worked/lasted/etc.]"

Paragraph 3: The turning point
"Then [discovery/realization/meeting] changed everything..."

Paragraph 4: The result
"Now [specific outcome with numbers or timeframe]"

Paragraph 5: The bridge
"I realized others were struggling with the same thing, so..."

---

[OFFER: Clear, compelling, low-friction]

"Here's what I put together: [Product/Service name]"

What's included:
• [Component 1 + benefit]
• [Component 2 + benefit]
• [Component 3 + benefit]

[Price framing if applicable]

[Urgency if authentic]

---

[CTA: Single, clear action]

[Link or button]

[P.S.: Restate key benefit or add bonus/deadline]
```

## 3. Product Description Template

**Source:** Bly's Features-Benefits Spectrum (BRAND-B2B § 2.2)

```
[HEADLINE: Benefit-driven, not feature-driven]

[OPENING LINE: The problem this solves or desire it fulfills]

---

THE PRODUCT:
[One sentence describing what it is]

---

WHY IT MATTERS:

[Feature 1] → [Advantage over alternatives] → [Benefit to user] → [Ultimate benefit: status/identity/emotion]

[Feature 2] → [Advantage over alternatives] → [Benefit to user] → [Ultimate benefit: status/identity/emotion]

[Feature 3] → [Advantage over alternatives] → [Benefit to user] → [Ultimate benefit: status/identity/emotion]

---

SPECS (if applicable):
• [Spec 1]
• [Spec 2]
• [Spec 3]

---

[CLOSING LINE: Transformation or identity statement]

[CTA]
```

**Fill-in example:**

```
Feature: 40-hour battery
Advantage: Longest in category (competitors average 20 hours)
Benefit: Charge once a week instead of daily
Ultimate: Never be the person scrambling for an outlet in a meeting
```

## 4. Headline Templates by Awareness Level

**Source:** Schwartz's Five Levels (CLASSIC § 2.1)

### Most Aware (knows product, ready to buy)
```
[Product name]: Now [improvement/discount/bonus]
[X]% off [Product] — [deadline]
[Product] + [Bonus] — [limited time]
```

### Product Aware (knows product, not convinced)
```
Why [X number] [audience] switched to [Product]
[Product] vs. [Alternative]: [specific difference]
The [feature] that [outcome] — see how [Product] does it
```

### Solution Aware (knows solutions exist, not your product)
```
The [adjective] way to [desired outcome]
[Outcome] without [pain point]
How to [achieve result] in [timeframe]
```

### Problem Aware (knows problem, not solutions)
```
Tired of [problem]?
[Problem] is costing you [consequence]
Why [problem] keeps happening (and what to do about it)
```

### Unaware (doesn't know they have a problem)
```
[Surprising statistic about their situation]
[X]% of [audience] are [doing thing wrong] — are you?
What [successful people] know about [topic] that you don't
```

## 6. Urgency/Scarcity Section Template

**Source:** Cialdini's Scarcity (PSYCHOLOGY § 1.2) + Kahneman's Loss Aversion (PSYCHOLOGY § 2.1)

```
═══════════════════════════════════════════════════════════
QUANTITY SCARCITY (if authentic)
═══════════════════════════════════════════════════════════

"Only [X] spots available"
"[X] remaining at this price"
"Limited to [X] per [time period]"

+ Reason why: "[Because we personally onboard each customer]"

═══════════════════════════════════════════════════════════
TIME SCARCITY (if authentic)
═══════════════════════════════════════════════════════════

"Available until [specific date]"
"Price increases [date] at [time]"
"Enrollment closes in [countdown]"

+ What they lose: "After [date], [consequence]"

═══════════════════════════════════════════════════════════
LOSS FRAMING
═══════════════════════════════════════════════════════════

Instead of: "Get [benefit]"
Write: "Don't miss [benefit]" or "Stop losing [thing]"

Instead of: "Save $X"
Write: "Stop wasting $X on [inferior alternative]"

Instead of: "Join X members"
Write: "Don't be the only one still [doing old thing]"
```

## 8. Risk Reversal Section Template

**Source:** Wiebe's Zero Risk Sweep (MODERN-DIGITAL § 1.4) + Sugarman's Triggers (PSYCHOLOGY § 3.3)

```
═══════════════════════════════════════════════════════════
GUARANTEE FORMULAS
═══════════════════════════════════════════════════════════

Basic:
"[X]-day money-back guarantee. No questions asked."

Specific:
"Try [Product] for [X] days. If [specific condition], [specific remedy]."

Bold:
"If you don't [achieve specific result] in [timeframe], [remedy + extra]."

═══════════════════════════════════════════════════════════
OBJECTION HANDLING
═══════════════════════════════════════════════════════════

Format:
"Worried about [objection]? [Direct answer]"

Common objections to address:
• "What if it doesn't work for my situation?"
• "What if I don't have time to implement?"
• "What if I need help?"
• "What's the catch?"
• "Why is it priced this way?"

═══════════════════════════════════════════════════════════
THE FIVE RISKS TO ADDRESS
═══════════════════════════════════════════════════════════

1. Performance risk: "Will it work?"
   → [Guarantee, case studies, demo]

2. Value risk: "Is it worth the money?"
   → [ROI calculation, price comparison, payment plans]

3. Design risk: "Will it fit my situation?"
   → [Use cases, customization, compatibility]

4. Social risk: "Will others judge me?"
   → [Testimonials from similar others, privacy assurance]

5. Joy risk: "Will I regret this?"
   → [Trial period, easy cancellation, satisfaction stats]
```

## 10. Epiphany Bridge Story Template

**Source:** Brunson's Epiphany Bridge (MODERN-DIGITAL § 2.4)

```
═══════════════════════════════════════════════════════════
STEP 1: THE INTRODUCTION (Drop into the struggle)
═══════════════════════════════════════════════════════════

"[Time marker], I was [struggling with specific problem]."

"I had tried [Solution A], [Solution B], and [Solution C]."

"Nothing was working. I felt [emotion]."

═══════════════════════════════════════════════════════════
STEP 2: THE JOURNEY & CONFLICT
═══════════════════════════════════════════════════════════

"What made it worse was [consequence or escalation]."

"I remember thinking [internal dialogue showing desperation]."

"Then [triggering event that started the journey]."

═══════════════════════════════════════════════════════════
STEP 3: THE NEW OPPORTUNITY (The epiphany)
═══════════════════════════════════════════════════════════

"That's when I realized [insight that changed everything]."

"It wasn't about [old belief]. It was about [new belief]."

"This completely changed how I thought about [topic]."

═══════════════════════════════════════════════════════════
STEP 4: THE FRAMEWORK (High-level only, no details)
═══════════════════════════════════════════════════════════

"I developed a [simple/3-step/new] approach:"

"First, [high-level step 1]."
"Then, [high-level step 2]."
"Finally, [high-level step 3]."

(Do NOT give enough detail to implement. Create curiosity gap.)

═══════════════════════════════════════════════════════════
STEP 5: THE RESULTS
═══════════════════════════════════════════════════════════

"Within [timeframe], I [achieved specific measurable result]."

"But it wasn't just me. [Person/group] used this and [their result]."

"And [another example] got [their result]."

═══════════════════════════════════════════════════════════
BRIDGE TO OFFER
═══════════════════════════════════════════════════════════

"I realized I had to share this."

"That's why I created [Product/Offer]."

[Transition to offer details]
```

## Usage Notes

**Templates are starting points, not formulas.** Adapt structure to fit your specific:
- Audience awareness level
- Market sophistication stage
- Brand voice requirements
- Content length constraints

**Always validate against Human-Check Protocol** before publishing. Templates can inadvertently create robotic-sounding copy if filled in mechanically.

**Source references** point to detailed application rules. When in doubt, consult the original framework documentation.
